Research Buddy is a **full-stack AI assistant** for university students that makes dense academic papers accessible. Paste in a paper вҖ” get back a plain-English summary, the key topics, and a chat interface to ask specific questions, all grounded in the actual paper text.

```bash

cd backend

python -m venv venv
```

## рҹ“Ў API Reference

### `GET /`
Health check вҖ” confirms the backend is alive.

**Response:**
```json
{
  "status": "ok",
  "message": "Research Buddy API is running"
}
```

---

### `POST /upload`
Accepts a PDF file, extracts text and metadata using PyMuPDF.

**Request:** `multipart/form-data` with field `file` containing the PDF.

**Response:**
```json
{
  "status": "success",
  "filename": "attention_is_all_you_need.pdf",
  "title_guess": "Attention Is All You Need",
  "page_count": 15,
  "word_count": 9241,
  "raw_text": "Attention Is All You Need\nAbstract. The dominant sequence...",
  "full_text": "< complete extracted text of all pages >"
}
```

**Error responses:**

| Code | Reason |
|------|--------|
| `400` | Uploaded file is not a PDF (wrong MIME type) |
| `422` | File is corrupt or cannot be parsed by PyMuPDF |

## рҹ“Ӣ .gitignore Placement Guide

```
research-buddy/
в”ңв”Җв”Җ .gitignore     вҶҗ вң… Place HERE (project root вҖ” covers both backend & frontend)
в”ңв”Җв”Җ backend/
в””в”Җв”Җ frontend/
```

**What it ignores and why:**

| Pattern | Reason |
|---------|--------|
| `venv/` | Python virtual environment вҖ” gigabytes, rebuilt by each developer |
| `__pycache__/` | Python bytecode вҖ” auto-generated, not source code |
| `.env` | Environment variables вҖ” contains secrets and API keys |
| `node_modules/` | npm packages вҖ” hundreds of MB, restored via `npm install` |
| `frontend/dist/` | Vite build output вҖ” generated artifact, not source |
| `*.pdf` | Uploaded user files вҖ” user data should never be in git |
| `.DS_Store` | macOS Finder metadata вҖ” machine-specific noise |
| `chroma_db/` | Phase 3+ vector database вҖ” large binary files |
